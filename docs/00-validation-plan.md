## 1. 목적

본 계획은 `pharma-ntp-standard` 시스템의 컴퓨터 시스템 검증(CSV) 접근 방식, 범위, 책임, 산출물, 합격 기준을 정의한다.

## 2. 시스템 개요

제약(GxP) 환경에서 NTP 시간 동기화 표준을 수립·모니터링·검증하는 웹 애플리케이션. GxP 전자 기록의 타임스탬프 신뢰성을 보장하는 것이 목적이다.

## 3. GAMP 5 분류 및 시스템 영향도 평가

### 3.1 GAMP 5 소프트웨어 카테고리

본 시스템은 **GAMP 카테고리 5(맞춤 개발 애플리케이션)** 로 분류한다. 표준 NTP 프로토콜(시간 소스)은 인프라(카테고리 1)에 의존하나, 모니터링·검증·표준 관리 로직은 맞춤 개발이다.

### 3.2 시스템 영향도 평가 (System Impact Assessment)

본 시스템은 **직접 영향 시스템(Direct Impact System)** 으로 분류한다. 최신 분류 체계는 '간접 영향(Indirect Impact)'을 별도 등급으로 두지 않으며, GxP 데이터 무결성에 영향을 주는 시스템은 직접 영향으로 본다.

- **근거**: 본 시스템이 생성하는 [한계 초과 로그(FS-023)](02-functional-spec.md)·[정합성 보고서(VSR-001)](10-kriss-conformance-report.md)는 GxP 전자기록 타임스탬프의 신뢰성을 입증·판단하는 근거로 사용된다. 경고 판정([RISK-001](04-risk-assessment.md))이 오작동하면 비동기 장비가 검출되지 않아, 해당 장비가 남긴 GxP 기록의 시각 무결성(ALCOA+의 *Contemporaneous*)이 훼손될 수 있다.
- **범위 경계(중요)**: 본 시스템은 장비 시계를 **설정·보정하지 않는다** — 감시·읽기 전용(NTP mode 3 클라이언트 한정, [RISK-011](04-risk-assessment.md)). 실제 시각 동기화(보정)는 인프라(NTP 서버·OS, GAMP 카테고리 1)의 책임으로 본 검증 범위 밖이다. 따라서 직접 영향의 대상은 **데이터 무결성(감시·보고 기록)** 이며, 시각 *제어* 기능은 시스템에 포함되지 않는다.
- **결과**: 직접 영향 분류에 따라 본 시스템은 전체 V 모델 검증(IQ/OQ/PQ) 대상이다. 위험 기반으로 GxP 영향도가 높은 기능(오프셋 경고·표준 무결성, RISK-001/002)에 검증 노력을 집중한다(§5 참조).

## 4. 검증 접근(V 모델)

| 단계 | 산출물 | 합격 기준 |
|------|--------|----------|
| 요구사항 | URS | 모든 요구사항이 검증 가능(testable)하고 추적 가능 |
| 기능 명세 | FS | 모든 URS 항목이 1개 이상의 FS로 전개 |
| 설계 명세 | DS | 모든 FS 항목이 설계로 구현 가능 |
| 설치 검증 | IQ | 설치/구성 항목 100% 통과 |
| 운영 검증 | OQ | 기능 테스트 케이스 100% 통과(또는 일탈 승인) |
| 성능 검증 | PQ | 실사용 시나리오에서 요구 성능 충족 |

## 5. 위험 기반 접근

[위험 평가(RISK)](04-risk-assessment.md)에 따라 GxP 영향도가 높은 기능(시간 오프셋 경고, 표준 무결성)에 검증 노력을 집중한다.

## 6. 역할과 책임

| 역할 | 책임 |
|------|------|
| 제품 소유자 | URS 승인, PQ 승인 |
| 개발자 | FS/DS 작성, 구현, 단위 테스트 |
| 품질(QA) | 산출물 검토·승인, 일탈 관리 |
| 검증 담당 | IQ/OQ/PQ 실행·기록 |
