pharma-ntp-standard 사용 방법

국가표준시 KRISS UTC(k)(time.kriss.re.kr)를 기준으로 장비 시계의 오차를 감시하는 도구입니다.

읽기 전용 감시 — 장비 시각을 바꾸지 않습니다

이 문서는 실제 사용에 필요한 것만 담았습니다(설치·검증 문서는 별도). 처음이라면 위에서 아래로 한 번 훑어보세요.

1실행과 종료

  1. pharma-ntp-standard.exe를 더블클릭합니다.
  2. 검은 콘솔 창이 뜨고, 잠시 후 브라우저가 자동으로 화면(http://localhost:8000)을 엽니다.
  3. 브라우저가 안 열리면 주소창에 직접 http://localhost:8000 을 입력하세요.
  4. 종료하려면 콘솔 창에서 Ctrl + C 를 누르거나 창을 닫습니다.
알아두기 콘솔 창 = 프로그램 본체입니다. 감시를 계속하려면 이 창을 켜둔 채로 두세요(닫으면 측정이 멈춥니다). 설정·장비·기록 같은 데이터는 C:\ProgramData\PharmaNTP\ 에 저장되어, 프로그램 폴더를 옮기거나 새 버전으로 바꿔도 유지됩니다.

2화면 한눈에

영역역할
대형 시계KRISS 기준 현재시각. 초록 동기화 양호 / 주황 미검증(기준 도달 실패). 전체화면 버튼 제공.
동기화 모니터링 대시보드등록한 장비별 오차(오프셋)·상태·마지막 측정 시각. 자동으로 주기 갱신됩니다.
허용 한계 초과 로그오차가 허용 한계를 벗어난 기록(최근 1주일 / 지난 이력 조회).
동기화 설정 · 장비 등록 · 화면 설명평소 접혀 있는 카드. 헤더를 클릭하면 펼쳐집니다.

3장비 등록 (핵심)

감시할 장비를 추가하는 방법입니다. 처음 실행 시 기본 표준(KRISS UTC(k) 표준)이 이미 들어 있어 바로 등록할 수 있습니다.

먼저 — 장비가 'NTP 응답'을 해야 합니다 이 시스템은 장비에게 "지금 몇 시냐"를 NTP로 물어봐서 오차를 잽니다. 따라서 대상 장비에서 NTP 서버(응답) 기능이 켜져 있고, 방화벽이 UDP 123 을 허용해야 합니다. (응답하지 않는 장비는 등록이 거부됩니다 — 죽은 장비가 등록부를 오염시키지 않게 하는 안전장치입니다.) 장비 쪽 설정은 담당자/장비 매뉴얼을 따르세요.

등록 절차

  1. 화면 아래 “장비 등록” 카드 헤더를 클릭해 펼칩니다.
  2. 장비 이름(예: HPLC-WS-01), IP 또는 호스트(예: 10.0.0.5), 표준(기본 KRISS UTC(k) 표준)을 입력·선택합니다.
  3. [추가] 를 누릅니다. 시스템이 그 IP로 NTP 응답을 즉시 확인합니다.
  4. 응답하면 등록 완료 → 대시보드에 행이 생기고 자동으로 주기 측정이 시작됩니다.
등록이 거부될 때 — 원인 “NTP 응답을 하지 않아 등록할 수 없습니다” 가 뜨면 대상 장비가 NTP 질의에 답하지 않는 것입니다(IP가 살아 있어도, ping이 돼도 발생). 원인은 보통 ① 장비의 NTP 서버(응답) 기능이 꺼짐, ② 방화벽이 UDP 123 차단, 둘 중 하나입니다.

조치 방법 — 장비가 NTP 응답을 하게 만들기

가장 쉬운 길 — 응답하는 대상을 등록

개별 PC 대신 사내 NTP 서버·도메인 컨트롤러(DC) 의 IP를 등록하세요. 그쪽은 보통 NTP 응답이 켜져 있어 추가 설정 없이 바로 등록됩니다. (장비들도 결국 그 상위 시간원에 맞추므로 상위를 감시하는 편이 효율적입니다.)

특정 Windows 장비를 꼭 감시해야 한다면 (그 장비에서, 관리자 권한)

① NTP 서버(응답) 기능 켜기 + 서비스 재시작:

reg add "HKLM\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\NtpServer" /v Enabled /t REG_DWORD /d 1 /f
reg add "HKLM\SYSTEM\CurrentControlSet\Services\W32Time\Config" /v AnnounceFlags /t REG_DWORD /d 5 /f
net stop w32time && net start w32time

② 방화벽 인바운드 UDP 123 허용:

netsh advfirewall firewall add rule name="NTP-in" dir=in action=allow protocol=UDP localport=123

③ 감시 PC(이 프로그램이 도는 PC)에서 응답 확인 — 오프셋 숫자가 나오면 정상:

w32tm /stripchart /computer:<장비IP> /dataonly /samples:3

확인되면 화면에서 [추가] 를 다시 누르면 등록됩니다.

Linux · 계측기 · 임베디드 장비 Linux는 /etc/chrony/chrony.conf 에 allow <감시PC 대역> 을 넣고 sudo systemctl restart chronyd. 계측기·네트워크 장비는 벤더 설정 화면의 NTP 서버(응답) 항목을 켭니다. 응답을 지원하지 않는 장비는 직접 감시할 수 없으니, 그 장비가 동기화하는 상위 NTP 서버를 대신 감시하세요.

미측정아직 1회도 측정 전다음 측정 주기까지 대기

5설정 바꾸기 (선택)

“동기화 설정” 카드에서 폴링 주기(초) 와 허용 한계(초) 를 조정할 수 있습니다. 변경 시 사유를 함께 적으면 표준 변경 이력에 기록됩니다(누가·언제·무엇을·왜).

권장 출발점 허용 한계는 운영 요구에 맞춰 정하세요(시각 동기 기준은 보통 수 초 이내로 빡빡하게 둡니다). 폴링 주기는 60~300초 정도가 일반적입니다.

6자주 묻는 질문

시계가 주황색(미검증)으로 떠요.
KRISS(time.kriss.re.kr) 도달에 실패한 상태입니다(인터넷·방화벽). 이때는 신뢰할 수 없는 서버 시각을 임시 표시합니다. 네트워크가 회복되면 다시 초록으로 바뀝니다.
장비 등록이 계속 거부돼요.
대상 장비의 NTP 서버 응답이 꺼져 있거나 방화벽 UDP 123 이 막힌 경우가 대부분입니다. 장비 쪽 설정을 켜거나, 응답하는 사내 NTP/DC 를 대신 등록하세요.
데이터를 백업하려면?
C:\ProgramData\PharmaNTP\pharma_ntp.sqlite3 파일을 복사해 두면 됩니다(표준·장비·기록 전부 포함).
실행했더니 포트 오류가 나요.
다른 프로그램이 이미 8000 포트를 쓰는 경우입니다. 그 프로그램을 끄거나, (고급) 환경변수 NTP_PORT 로 다른 포트를 지정해 실행하세요.
이 프로그램은 장비 시각을 변경하지 않는 읽기 전용 감시 도구이며, 공식 인증 제품이 아니라 KRISS UTC(k)에 준하는 참조 구현입니다. 실제 규제(GxP) 환경 적용 시에는 사용 기관의 별도 검증(밸리데이션)이 필요합니다.
