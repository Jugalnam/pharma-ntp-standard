# 성능 적격성 (Performance Qualification, PQ)

| 항목 | 내용 |
|------|------|
| 문서 ID | PQ-001 |
| 버전 | 0.1 (Draft) |
| V 모델 대응 | 사용자 요구사항(URS) |
| 전제 | OQ 통과 |

## 1. 목적

실사용에 준하는 조건에서 시스템이 [URS](01-user-requirements.md)를 충족함을 입증한다. 즉 "사용자가 의도한 대로, 운영 환경에서" 동작하는지 확인한다.

## 2. 테스트 케이스

| ID | 검증 요구(URS) | 시나리오 | 합격 기준 | 결과 |
|----|---------------|---------|----------|------|
| PQ-001 | 표준 운영 (URS-001/002) | KRISS UTC(k) 기준 표준을 실제 NTP 소스(`time.kriss.re.kr`)로 구성·운영 | 실 소스에서 오프셋 수집 성공, UTCk 시각과 차이 ≤ 한계 | ☐ |
| PQ-002 | 지속 모니터링 (URS-020) | 대표 장비를 24시간 연속 폴링 | 폴링 누락 없음, 드리프트 추세 기록됨 | ☐ |
| PQ-003 | 대시보드 성능 (URS-021) | 장비 500대 시뮬레이션 후 대시보드 조회 | 응답 ≤ 1초 | ☐ |
| PQ-004 | 한계 초과 운영 대응 (URS-022) | 실제 장비 시각을 인위적 이탈 | 경고 발생·통지, 운영자 인지 가능 | ☐ |
| PQ-005 | 검증 산출물 흐름 (URS-030/031) | IQ→OQ→PQ 산출물을 실제 승인 흐름으로 처리 | 상태 전이·승인 이력 정상 | ☐ |

## 3. UTCk 정합성 확인(핵심)

PQ-001에서 본 시스템이 보고하는 기준 시각과 **공식 프로그램 UTCk가 표시하는 KRISS 시각**을 동시 측정·비교한다.
- 합격: 두 시각의 차이가 표준 허용 한계 이내
- 본 비교로 "공식 UTCk에 *준하는*" 시각 기준을 사용함을 입증한다.

## 4. 합격 기준

모든 PQ 케이스 통과. 미해결 중대 일탈 없음 → 시스템 릴리스 가능([VP](00-validation-plan.md) §8).
