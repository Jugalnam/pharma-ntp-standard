# 사용자 요구사항 명세 (User Requirements Specification, URS)

| 항목 | 내용 |
|------|------|
| 문서 ID | URS-001 |
| 버전 | 0.1 (Draft) |
| V 모델 대응 | 성능 적격성(PQ) |

## 1. 개요

본 문서는 `pharma-ntp-standard` 시스템에 대한 사용자(품질·운영) 관점의 요구사항을 정의한다.
각 요구사항은 검증 가능해야 하며, [PQ](09-pq-protocol.md)에서 입증된다.

### 1.1 기준 표준 (Reference Standard)

본 시스템의 신뢰 시간 기준은 **한국표준과학연구원(KRISS)이 유지하는 협정세계시 실현 UTC(KRISS)** 이며,
KRISS 공식 시각 동기화 프로그램 **UTCk**(v4.1.0.2, `time.kriss.re.kr` 계열 NTP 소스)가 제공하는 시각을 표준 기준점으로 삼는다.
본 프로젝트는 공식 인증 제품이 아니며, UTCk를 **참조 기준**으로 하여 그에 준하는 모니터링·검증 체계를 제공한다.

## 2. 요구사항 표기

- **GxP 영향도**: H(높음)/M(중간)/L(낮음) — [위험 평가](04-risk-assessment.md)와 연계
- **우선순위**: Must / Should / Could

## 3. 표준 정의 (Time Standard)

| ID | 요구사항 | GxP | 우선순위 |
|----|----------|-----|---------|
| URS-001 | 시스템은 **KRISS UTC(k)** 를 신뢰 시간 기준으로 하여(기본 NTP 소스: `time.kriss.re.kr`), 이를 시간 표준으로 정의·관리할 수 있어야 한다. | H | Must |
| URS-002 | 시스템은 동기화 표준 파라미터(허용 오프셋 한계, 폴링 주기, 재동기 정책)를 정의할 수 있어야 한다. | H | Must |
| URS-003 | 표준은 버전 관리되어야 하며, 변경 시 사유와 승인 이력이 남아야 한다. | H | Must |

## 4. 장비/시스템 관리 (Asset Management)

| ID | 요구사항 | GxP | 우선순위 |
|----|----------|-----|---------|
| URS-010 | 시스템은 시간 동기화 대상 장비/시스템을 등록·조회·수정할 수 있어야 한다. | M | Must |
| URS-011 | 각 장비는 GxP 중요도(critical 여부)로 분류될 수 있어야 한다. | H | Must |
| URS-012 | 장비에 동기화 표준(정책)을 할당할 수 있어야 한다. | M | Should |

## 5. 모니터링 (Monitoring)

| ID | 요구사항 | GxP | 우선순위 |
|----|----------|-----|---------|
| URS-020 | 시스템은 등록 장비의 시간 오프셋(표준 대비 편차)을 주기적으로 수집해야 한다. | H | Must |
| URS-021 | 시스템은 각 장비의 오프셋·드리프트·stratum·마지막 동기 시각을 대시보드로 표시해야 한다. | M | Must |
| URS-022 | 오프셋이 허용 한계를 초과하면 시스템은 경고(alert)를 발생시켜야 한다. | H | Must |
| URS-023 | 경고 발생·해제 이력은 보존되어야 한다. | H | Should |

## 6. 검증 산출물 관리 (Validation Deliverables)

| ID | 요구사항 | GxP | 우선순위 |
|----|----------|-----|---------|
| URS-030 | 시스템은 IQ/OQ/PQ 검증 산출물을 생성·조회·갱신할 수 있어야 한다. | M | Must |
| URS-031 | 검증 산출물은 상태(Draft→Approved→Effective)를 가지며 승인 워크플로우를 따라야 한다. | H | Should |
| URS-032 | 산출물은 관련 요구사항(URS/FS/DS)에 추적 연결될 수 있어야 한다. | M | Could |

## 7. 데이터 무결성 / 규제 (전제)

| ID | 요구사항 | GxP | 우선순위 |
|----|----------|-----|---------|
| URS-040 | 모든 중요 데이터 변경은 변경 불가한 감사 추적(누가/언제/무엇을/왜)으로 기록되어야 한다. (21 CFR Part 11) | H | Should |
| URS-041 | 시스템 자체의 타임스탬프는 신뢰 시간 소스에 동기화되어야 한다. | H | Must |

> 주: URS-040(감사 추적)·승인 워크플로우는 초기 범위에서는 기반 설계만 두고 후속 반복에서 구현한다.
